* tmux Configuration
:PROPERTIES:
:ID:       a1f3c8d2-7e4b-4a91-b5e0-8c2d1f6a9b3e
:END:
This is my personal configuration for tmux. It's written using Org Mode's literate programming features,
which allows embedding documentation directly alongside the code. This file is configured to automatically
"tangle" the code blocks into the actual tmux configuration file (named =tmux.conf= in the same directory)
whenever this Org file is saved in Emacs (see the [[*Setting Up Auto-Tangle][Setting Up Auto-Tangle]] section). This approach keeps the
configuration well-documented and easier to manage.

The config lives at =~/.config/tmux/tmux.conf= (XDG-compliant). Plugins are managed by TPM (Tmux Plugin
Manager) and installed under =~/.config/tmux/plugins/=.

** Installation
:PROPERTIES:
:ID:       a5b7c9d1-e3f5-4a7b-8c9d-2e4f6a8b0c1d
:END:
Steps to get this configuration up and running from scratch.

*** Install TPM
:PROPERTIES:
:ID:       b6c8d0e2-f4a6-4b8c-8d0e-3f5a7b9c1d2e
:END:
Clone the Tmux Plugin Manager into the XDG-compliant plugins directory:

#+begin_example
git clone https://github.com/tmux-plugins/tpm ~/.config/tmux/plugins/tpm
#+end_example

*** Tangle the Config
:PROPERTIES:
:ID:       c7d9e1f3-a5b7-4c9d-8e1f-4a6b8c0d2e3f
:END:
If you have auto-tangle set up (see [[*Setting Up Auto-Tangle][Setting Up Auto-Tangle]]), just save this file in Emacs and the
=tmux.conf= will be generated automatically. Otherwise, open this file in Emacs and run:

#+begin_example
M-x org-babel-tangle
#+end_example

This produces =tmux.conf= in the same directory as this =README.org= file. Make sure both live inside
=~/.config/tmux/=.

*** Install Plugins
:PROPERTIES:
:ID:       d8e0f2a4-b6c8-4d0e-8f2a-5b7c9d1e3f4a
:END:
Once the config is tangled and TPM is cloned:

1. Start a new tmux session (or reload the config in an existing one with =C-Space r=).
2. Press =C-Space I= (capital I). TPM will clone and install all plugins listed in the config.
3. You should see a message confirming the installation. Press =Enter= to dismiss it.

To update plugins later, press =C-Space U= (capital U). To remove plugins you've deleted from the
config, press =C-Space alt-u=.

** Getting Started
:PROPERTIES:
:ID:       a2b4c6d8-e0f2-4a3b-8c5d-7e9f1a2b4c6d
:END:
A quick primer on how tmux works and the keybindings in this config.

*** Core Concepts
:PROPERTIES:
:ID:       b3c5d7e9-f1a3-4b5c-8d7e-0f2a4b6c8d0e
:END:
tmux organizes things in three layers:

- *Sessions*: Top-level containers. You might have one per project ("my-project", "yams", "dotfiles"). Sessions
  persist even if you close your terminal; reattach with =tmux attach -t name=.
- *Windows*: Tabs within a session. Shown in the status bar at the bottom. Switch with =C-b 1=, =C-b 2=,
  etc.
- *Panes*: Splits within a window. A single window can have multiple panes side by side or stacked.

The *prefix key* is =C-b= (the default). Every tmux command starts with the prefix, then a second key.
For example, =C-b c= means: press =Ctrl+b=, release, then press =c=.

*** Workflow Example
:PROPERTIES:
:ID:       c4d6e8f0-a2b4-4c6d-8e8f-1a3b5c7d9e1f
:END:
A typical session might look like this:

1. Start tmux: =tmux new -s myproject=
2. You're in window 1. Open a split: =C-b |= (vertical) or =C-b -= (horizontal).
3. Navigate between panes: =C-b h/j/k/l=.
4. Open a new window: =C-b c=. Now you have two tabs in the status bar.
5. Switch between windows: =C-b 1=, =C-b 2=.
6. Detach (leave everything running): =C-b d=.
7. Come back later: =tmux attach -t myproject=. Everything is exactly where you left it.

*** Quick Reference
:PROPERTIES:
:ID:       d5e7f9a1-b3c5-4d7e-8f9a-2b4c6d8e0f2a
:END:
Cheat sheet for all keybindings defined in this config.

#+begin_example
PREFIX = C-b

Sessions:
  C-b d             Detach from session
  C-b $             Rename session
  C-b s             List sessions (built-in)

Windows:
  C-b c             New window (in current dir)
  C-b 1-9           Switch to window N
  C-b ,             Rename window
  C-b &             Close window
  C-b n / p         Next / previous window (built-in)

Panes:
  C-b |             Split vertically
  C-b -             Split horizontally
  C-b h/j/k/l       Navigate panes
  C-b H/J/K/L       Resize panes (repeatable)
  C-b x             Close pane (built-in)
  C-b z             Toggle pane zoom/fullscreen (built-in)
  C-b q             Show pane numbers, press number to jump (built-in)

Copy Mode:
  C-b [             Enter copy mode (navigate with vim keys)
  v                 Start selection (in copy mode)
  y                 Yank to system clipboard (in copy mode)
  q                 Exit copy mode

Management:
  C-b r             Reload config
  C-b I             Install plugins (TPM)
  C-b U             Update plugins (TPM)
#+end_example
** Philosophy
:PROPERTIES:
:ID:       b2e4a7c1-3d5f-4b82-a6c9-1e7f2d8b0a4c

understood, and modified. Doom Emacs is the IDE; this is the terminal multiplexer. The goals are:

- Vim-style navigation
- Minimal plugins (only what earns its place)

** Terminal & Color
:PROPERTIES:
:ID:       d4a6c9e3-5f7b-6da4-c8e1-3a9b4f0d2c6e
:END:
True color support is essential for Neovim themes (like carbonfox) to render correctly inside tmux.
Without these settings, colors fall back to 256-color approximations and look washed out.
- =default-terminal=: Sets the =$TERM= value inside tmux sessions. =tmux-256color= is the correct terminfo
  entry for modern tmux.
- =terminal-features=: Appends RGB (true color) capability to =xterm-256color= terminals. This is the
  modern way to enable 24-bit color in tmux 3.2+.

If colors still look wrong, check that your terminal emulator (alacritty, ghostty, kitty, etc.) also
reports true color support.
#+begin_src conf
# True color support
set -g default-terminal "tmux-256color"
set -as terminal-features ",xterm-256color:RGB"
#+end_src

** General Settings
:PROPERTIES:
:ID:       e5b7daf4-6a8c-7eb5-d9f2-4b0c5a1e3d7f
:END:
Core settings that make tmux behave sanely.

*** Escape Time
:PROPERTIES:
:ID:       f6c8eb05-7b9d-8fc6-ea03-5c1d6b2f4e8a
:END:
When you press Escape, tmux waits to see if it's the start of a terminal escape sequence (like an arrow
key code). The default wait is 500ms, which makes Neovim feel sluggish when leaving insert mode with
Escape. Setting this to 0 makes mode switching instant. If you ever see weird behavior with function keys
or arrow keys, bump this to 10.
#+begin_src conf
# Eliminate ESC delay (critical for nvim inside tmux)
set -sg escape-time 0
#+end_src

*** Scrollback Buffer
:PROPERTIES:
:ID:       a7d9fc16-8cae-9ad7-fb14-6d2e7c3a5f9b
:END:
The scrollback buffer stores output history per pane. The default (2000 lines) runs out fast when tailing
logs or running builds. 50000 is generous without eating significant memory.
#+begin_src conf
# Bigger scrollback
set -g history-limit 50000
#+end_src

*** Indexing
:PROPERTIES:
:ID:       b8ea0d27-9dbf-0be8-ac25-7e3f8d4b6a0c
:END:
By default, windows and panes are numbered starting at 0. On a keyboard, 0 is on the far right while 1
is directly above the left hand. Starting at 1 makes =C-Space 1= reach the first window without
stretching. =renumber-windows= ensures that closing a middle window renumbers the rest sequentially (no
gaps like [1, 3, 4]).
#+begin_src conf
# Start numbering at 1 (0 is across the keyboard)
set -g base-index 1
setw -g pane-base-index 1
set -g renumber-windows on
#+end_src

*** Mouse Support
:PROPERTIES:
:ID:       c9fb1e38-0eca-1cf9-bd36-8f4a9e5c7b1d
:END:
Enables clicking to select panes, dragging to resize them, and scrolling with the mouse wheel. This is
purely additive; all keyboard workflows still work. Useful when you're lazy or forget a shortcut.
#+begin_src conf
# Mouse support
set -g mouse on

bind-key -T copy-mode-vi WheelUpPane send-keys -X -N 2 scroll-up
bind-key -T copy-mode-vi WheelDownPane send-keys -X -N 2 scroll-down
#+end_src

*** Focus Events
:PROPERTIES:
:ID:       d0ac2f49-1fdb-2da0-ce47-9a5b0f6d8c2e
:END:
Forwards terminal focus/unfocus events to programs running inside tmux. This lets Neovim detect when you
switch back to it and trigger =autoread= (reload files changed on disk). Without this, nvim inside tmux
won't know you tabbed away and came back.
#+begin_src conf
# Focus events (lets nvim detect focus for autoread, etc.)
set -g focus-events on
#+end_src

*** Session Destroy Behavior
:PROPERTIES:
:ID:       e1bd3a50-2aec-3eb1-df58-0b6c1a7e9d3f
:END:
Normally, destroying the last window in a session detaches you from tmux entirely, dumping you back to
the shell. With this off, tmux switches you to the next available session instead. Prevents accidental
disconnects when you run multiple sessions.
#+begin_src conf
# Don't detach when you destroy a session
set -g detach-on-destroy off
#+end_src

** Prefix Key
:PROPERTIES:
:ID:       f2ce4b61-3bfd-4fc2-ea69-1c7d2b8f0e4a
:END:
The prefix key is how you tell tmux "the next keypress is a tmux command, not input for the terminal
program." The default is =C-b=, and this config keeps it. While some people remap to =C-a= (from GNU
screen) or =C-Space=, the default is fine and means one less thing to explain when pair programming or
SSHing into a machine with a stock tmux.

The =send-prefix= binding means pressing =C-b C-b= sends a literal =C-b= through to the underlying
program, in case anything needs it (some shells use =C-b= for cursor movement).
#+begin_src conf
# Prefix: C-b (default, keeping it standard)
bind C-b send-prefix
#+end_src

*** Config Reload
:PROPERTIES:
:ID:       b4ea6d83-5dbf-6be4-ac81-3e9f4d0b2a6c
:END:
Reloads the tmux config without restarting. Essential while iterating on configuration. The =display=
part flashes a confirmation message so you know it worked.
#+begin_src conf
# Reload config
bind r source-file ~/.config/tmux/tmux.conf \; display "Config reloaded!"
#+end_src

*** Splits
:PROPERTIES:
:ID:       c5fb7e94-6eca-7cf5-bd92-4f0a5e1c3b7d
:END:
The default split bindings (="= and =%=) are unintuitive. These visual mnemonics are easier to remember:
=|= looks like a vertical divider, =-= looks like a horizontal one. Both open the new pane in the current
working directory instead of =$HOME=.

Note on tmux naming: =split-window -h= creates a horizontal split, meaning panes are side by side
(divided by a vertical line). It's backwards from what you'd expect, but the =|= binding makes the intent
clear.
#+begin_src conf
# Splits: | for vertical, - for horizontal (in current path)
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"
unbind '"'
unbind %
#+end_src

*** New Window
:PROPERTIES:
:ID:       d6ac8fa5-7fdb-8da6-cea3-5a1b6f2d4c8e
:END:
New windows open in the current directory instead of =$HOME=.
#+begin_src conf
# New window in current path
bind c new-window -c "#{pane_current_path}"
#+end_src

*** Pane Navigation
:PROPERTIES:
:ID:       e7bd9ab6-8aec-9eb7-dfb4-6b2c7a3e5d9f
:END:
Vim-style pane navigation using h/j/k/l. Muscle memory carries over from Neovim. The defaults (arrow
keys) still work; these are additions, not replacements.
#+begin_src conf
# Vim-style pane navigation
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R
#+end_src

*** Pane Resizing
:PROPERTIES:
:ID:       f8ceb0c7-9bfd-0fc8-eac5-7c3d8b4f6e0a
:END:
Same directions as navigation but uppercase for resizing. The =-r= flag means "repeatable": after
pressing =C-Space H= once, you can keep pressing =H= to keep resizing without hitting the prefix again
(within a short timeout window). The =5= is cells per press; adjust if it feels too coarse or too fine.
#+begin_src conf
# Resize panes with prefix + H/J/K/L (repeatable)
bind -r H resize-pane -L 5
bind -r J resize-pane -D 5
bind -r K resize-pane -U 5
bind -r L resize-pane -R 5
#+end_src

*** Copy Mode (Vi)
:PROPERTIES:
:ID:       a9dfb1d8-0cae-1ad9-fbd6-8d4e9c5a7f1b
:END:
Makes copy mode (entered with =C-Space [=) use vi-style navigation. Once in copy mode: =v= starts a
visual selection, =y= yanks it to the system clipboard (via the =tmux-yank= plugin). Without these
bindings, you'd use Space to start selection and Enter to copy, which is less intuitive for vim users.
#+begin_src conf
# Vi copy mode
setw -g mode-keys vi
bind -T copy-mode-vi v send-keys -X begin-selection
bind -T copy-mode-vi y send-keys -X copy-selection-and-cancel
#+end_src

** Status Bar
:PROPERTIES:
:ID:       b0eac2e9-1dbf-2be0-acd7-9e5f0d6b8a2c
:END:
Minimal status bar configuration. The actual theming is handled by the catppuccin plugin below. The
status bar sits at the bottom and refreshes every 5 seconds (default is 15). Don't go below 1 or you
waste CPU on redraws.
#+begin_src conf
# Status bar position and refresh
set -g status-position bottom
set -g status-interval 5
#+end_src

** Plugins
:PROPERTIES:

:END:
Plugins are managed by TPM (Tmux Plugin Manager). Install TPM first:
#+begin_example
git clone https://github.com/tmux-plugins/tpm ~/.config/tmux/plugins/tpm
#+end_example

After saving and tangling this file, open tmux and press =C-Space I= (capital I) to install all plugins.
Press =C-Space U= (capital U) to update them later.

*** Plugin Manager Path
:PROPERTIES:
:ID:       d2acd4a1-3fdb-4da2-cef9-1a7b2f8d0c4e
:END:
Tells TPM to install plugins under the XDG-compliant path instead of the default =~/.tmux/plugins/=.
#+begin_src conf
# XDG-compliant plugin path
set-environment -g TMUX_PLUGIN_MANAGER_PATH '~/.config/tmux/plugins'
#+end_src

*** Core Plugins
:PROPERTIES:
:ID:       e3bde5b2-4aec-5eb3-dfa0-2b8c3a9e1d5f
:END:
Three plugins, each earning its place:
- *tpm*: The plugin manager itself. Must be listed so it can manage its own updates.
- *tmux-sensible*: A set of "obviously correct" defaults that fill gaps without overriding anything set
  explicitly. Covers utf-8, better key handling, =emacs= keybindings in the command prompt (which is what
  you want even as a vim user; it means =C-a=/=C-e= work when typing tmux commands), and other small
  sanity fixes.
- *tmux-yank*: System clipboard integration for copy mode. When you yank text, it goes to the OS
  clipboard. On Linux it uses =xclip= or =xsel= (make sure one is installed). On macOS it uses =pbcopy=.
#+begin_src conf
# Plugins
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'
set -g @plugin 'tmux-plugins/tmux-yank'
#+end_src

*** Theme: Catppuccin
:PROPERTIES:
:ID:       f4cee6c3-5bfd-6fc4-eab1-3c9d4b0f2e6a
:END:
Catppuccin mocha provides a warm, dark color palette that pairs well with the carbonfox Neovim theme.
=rounded= gives the window tabs rounded separators using Nerd Font characters. The status line is
intentionally lean: nothing on the left, just the session name on the right.

To add more modules later, append to =status-right= using =set -ag=:
#+begin_example
set -ag status-right "#{E:@catppuccin_status_directory}"
set -ag status-right "#{E:@catppuccin_status_date_time}"
#+end_example

Available modules include: =application=, =session=, =directory=, =date_time=, =battery= (requires
=tmux-battery= plugin), =cpu= (requires =tmux-cpu= plugin), =uptime=, and others.
#+begin_src conf
# Catppuccin theme
set -g @plugin 'catppuccin/tmux#v2.1.3'
set -g @catppuccin_flavor 'mocha'
set -g @catppuccin_window_status_style 'rounded'

# Status line modules
set -g status-right-length 100
set -g status-left-length 100
set -g status-left ""
set -g status-right "#{E:@catppuccin_status_session}"
#+end_src

* Setting Up Auto-Tangle
:PROPERTIES:
:ID:       c7fbb9f6-8eca-9cf7-bde4-6f2a7e3c5b9d
:END:
To enable automatic tangling when this file is saved in Emacs, add the following to your Emacs
configuration. This checks for the =#+auto_tangle: t= property and runs =org-babel-tangle= on save.
If you already have this in your Emacs config from the i3 setup, you don't need to add it again.

#+begin_src emacs-lisp :tangle no
;; Auto-tangle configuration files
(use-package org
  :config
  (defun org-babel-auto-tangle ()
    "Automatically tangle org files when saved."
    (when (eq major-mode 'org-mode)
      (let ((auto-tangle (cdr (assoc "auto_tangle" (org-collect-keywords '("PROPERTY"))))))
        (when (and auto-tangle (string= auto-tangle "t"))
          (org-babel-tangle)))))
  (add-hook 'after-save-hook #'org-babel-auto-tangle))
#+end_src
